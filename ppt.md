# 블로그 프로젝트 구조 및 기능

---

## 1. 프로젝트 개요

- **목표**: 개인 기술 블로그와 포트폴리오를 결합한 웹사이트 구축
- **핵심 기술**: Python, Django, Bootstrap
- **주요 기능**: 블로그 포스팅, 프로젝트 쇼케이스, AI 기반 검색, 노트 기록

---

## 2. 핵심 기능 및 페이지 구조

### 2.1. 메인 블로그 (`/` 또는 `/blog`)

- **포스트 목록 (Post List)**
  - 모든 블로그 게시물을 최신순으로 보여주는 메인 페이지
  - 각 포스트는 제목, 요약 내용, 작성일 표시
  - 검색 기능 포함

- **포스트 상세 (Post Detail)**
  - 개별 포스트의 전체 내용을 보여주는 페이지
  - Markdown 렌더링을 통해 코드 하이라이팅 및 서식 지원

### 2.2. 프로젝트 포트폴리오 (`/projects`)

- **프로젝트 목록 (Project List)**
  - 진행했던 프로젝트들을 카드 형태로 보여주는 갤러리 페이지
  - 각 카드에는 프로젝트명, 간단한 설명, 사용 기술 태그 표시

- **프로젝트 상세 (Project Detail)**
  - **좌측 (메인 컨텐츠 영역)**
    - **개요 (About)**: 프로젝트에 대한 상세 설명
    - **기술 스택 (Technical Stack)**: 사용된 기술들의 아이콘과 이름 목록
    - **갤러리 (Gallery)**: 관련 스크린샷이나 이미지를 보여줄 공간 (추후 확장)
  - **우측 (사이드 정보 영역)**
    - 대표 이미지
    - 제작일
    - "Live Demo" 및 "Source Code" 바로가기 버튼

### 2.3. 추가 페이지

- **소개 (About)**: 블로그 주인에 대한 소개 페이지
- **연락 (Contact)**: 방문자가 연락을 남길 수 있는 이메일 폼
- **노트 (Notes)**: 간단한 아이디어나 메모를 기록하는 페이지
- **AI 검색 (AI Search)**: Gemini API를 활용한 시맨틱 검색 기능

---

## 3. 공통 UI 요소

- **네비게이션 바**: 모든 페이지 상단에 위치하며 주요 페이지로의 빠른 이동을 지원
- **사이드바**: 블로그 관련 페이지에 표시되며, 다음 정보를 포함:
  - 한 줄 소개
  - 주요 프로젝트 목록 (최대 4개) 및 사용 기술
  - 최근 글 목록
- **푸터**: 사이트 저작권 및 제작 정보 표시

---

## 4. 주요 기술 아키텍처

- **백엔드**: Django 프레임워크
  - MVC 패턴을 기반으로 한 안정적인 구조
- **프론트엔드**: Bootstrap 5
  - 반응형 웹 디자인을 위한 CSS 프레임워크
- **데이터베이스**: SQLite (개발용) / PostgreSQL (배포용)
- **커스텀 Django 기능**:
  - **Context Processors**: 사이드바에 표시될 최근 글, 프로젝트 목록 등을 모든 페이지에 공통으로 전달
  - **Custom Template Tags**: 기술 스택 이름을 Devicon 아이콘으로 변환하는 등 재사용 가능한 UI 로직 구현

---

## 5. 향후 개선 계획

- 프로젝트 상세 페이지의 갤러리 기능 활성화
- 사용자 인증 시스템 도입 (회원가입, 로그인)
- 포스트에 대한 댓글 및 좋아요 기능 추가
- CI/CD 파이프라인 구축을 통한 배포 자동화
